# 그리디 알고리즘
탐욕법
현재 상황에서 지금 당장 좋은 것만 고르는 방법
최소한의 아이디어 떠올릴 수 있는 능력 필요
일반적인 상황에서 그리디 알고리즘은 최적의 해를 보장할 수 없을 때가 많음
-> But, 코딩테스트 대부분의 그리디 문제는 탐욕법으로 얻은 해가 최적의 해가 되는 상황에서 이를 추론할 수 있어야 풀리도록 출제 됨.

ex) 거스름돈
최적의 해를 빠르게 구하기 위해서는 가장 큰 화폐 단위부터 돈을 거슬러 주면 됨

# 구현
- 방향벡터 (2차원 공간)
    | (0, 0) | (0, 1) | (0, 2) | (0, 3) | (0, 4) |
    | (1, 0) | (1, 1) | (1, 2) | (1, 3) | (1, 4) |
    | (2, 0) | (2, 1) | (2, 2) | (2, 3) | (2, 4) |
    | (3, 0) | (3, 1) | (3, 2) | (3, 3) | (3, 4) |
    | (4, 0) | (4, 1) | (4, 2) | (4, 3) | (4, 4) |
    
    ```sh
    # Python
    # 동, 북, 서. 남
    dx = [0, -1, 0, 1]
    dy = [1, 0, -1, 0]

    # 현재 위치
    x, y = 2, 2

    for i in range(4):
        # 다음 위치
        nx = x + dx[i]
        ny = y + dy[i]
        print(nx, ny)
    ```
